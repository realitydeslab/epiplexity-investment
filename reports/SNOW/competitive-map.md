# SNOW — Competitive Map

*As of March 7, 2026*

## Direct Competitors

### Databricks (Private)
- **Threat level:** HIGH
- **Overlap:** Data engineering, ML/AI workloads, lakehouse
- **Where Databricks wins:** Data engineering, streaming, ML model training, open-source ecosystem (Apache Spark, Delta Lake)
- **Where Snowflake wins:** SQL analytics, BI concurrency, data governance, ease of use, minimal tuning
- **Key insight:** The platforms are converging but from different directions. Databricks is adding SQL; Snowflake is adding AI/ML. For pure analytics, Snowflake is still superior. For heavy ML engineering, Databricks leads.
- **IPO risk:** If Databricks IPOs in 2026, it could draw investor attention/capital away from SNOW

### Cloud Hyperscalers (AWS, Azure, GCP)
- **Threat level:** MEDIUM
- **AWS Redshift, Azure Synapse, BigQuery** all compete on data warehousing
- **Snowflake's edge:** Multi-cloud neutrality, avoids vendor lock-in, superior cross-cloud data sharing
- **Risk:** Hyperscalers bundle analytics with compute at discount

### Datadog (DDOG)
- **Threat level:** MEDIUM (post-Observe acquisition)
- **Overlap:** Observability market now that Snowflake acquired Observe
- **Snowflake's edge:** Unified data + observability; Datadog is observability-only

### Palantir (PLTR)
- **Threat level:** LOW-MEDIUM
- **Overlap:** Enterprise AI platforms
- **Different approach:** Palantir is application-layer AI; Snowflake is data-infrastructure AI

## Who Loses if Snowflake Wins

| Company | Ticker | Why |
|---------|--------|-----|
| Teradata | TDC | Legacy data warehousing — migration target |
| Oracle | ORCL | On-prem database losing to cloud |
| Splunk (Cisco) | CSCO | Observe acquisition targets Splunk's market |
| Informatica | INFA | Data integration — Snowflake building native capabilities |

## Pair Trade Opportunities

1. **Long SNOW / Short TDC** — New cloud AI vs. legacy on-prem data warehousing
2. **Long SNOW / Long MDB** — Both benefit from enterprise AI data infrastructure buildout
3. **Long SNOW / Short DDOG (small)** — Observe integration could take share in observability
